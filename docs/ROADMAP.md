# Roadmap

## Phase 0

- Create governance documentation.
- Create recovery documentation.
- Define architecture and phase gates.

## Phase 1

- Scaffold backend, database, Prisma, and migrations.
- Implement core database entities.
- Create ERD and migration test plan.
- Define medicine normalization rules in executable form.

## Phase 2

- Implement source adapter interface.
- Implement audit logging.
- Add admin imports.
- Add DRAP and pharmacy source adapters.

## Phase 3

- Implement canonical signatures.
- Implement duplicate detection.
- Implement fuzzy matching.
- Add confidence scoring and review queues.

## Phase 4

- Build recommendation and price intelligence services.
- Build unknown medicine background discovery.
- Build prescription cost optimizer.

## Phase 5

- Build mobile-first PWA.
- Add medicine search, upload flows, reports, and dashboard.

## Phase 6

- Build admin application and review queues.
- Add crawler and analytics dashboards.

## Phase 7

- Implement verified pharmacy marketplace after legal and product validation.

## Phase 8

- Decide on warehouse expansion based on demand, savings, and operational evidence.
